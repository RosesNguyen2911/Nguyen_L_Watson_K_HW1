## ğŸ§  How It Works

1. Character data is fetched from the **SWAPI `/people` endpoint**
2. The API returns an array of character objects, which is limited to **12 characters** for layout consistency
3. Character cards are generated by **cloning an HTML `<template>`**, ensuring a consistent and reusable structure
4. Each character card stores related film URLs using `data-*` attributes
5. When a character is selected:
   - Each associated film URL is fetched individually
   - Movie details are rendered by cloning a movie template
   - The page scrolls smoothly to the movie section once all data is loaded
6. Loading and error states are handled gracefully throughout the process
